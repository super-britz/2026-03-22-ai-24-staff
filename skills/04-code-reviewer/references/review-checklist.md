# 代码质量检查清单

## 可读性（权重25%）
- [ ] 函数名清晰描述了它做什么
- [ ] 单一职责：每个函数只做一件事
- [ ] 函数长度不超过30行（不含注释）
- [ ] 圈复杂度不超过10
- [ ] 条件嵌套不超过3层
- [ ] 变量名有描述性（循环计数器除外不允许单字母变量）

## TypeScript 质量（权重25%）
- [ ] 没有使用 any 类型（不确定类型用 unknown 然后收窄）
- [ ] 所有函数参数和返回值都有显式类型标注
- [ ] 复杂对象使用 interface 或 type 定义
- [ ] 固定值集合使用联合类型或 as const
- [ ] 严格空值检查（没有无理由的非空断言 !）

## DRY 和模式（权重20%）
- [ ] 没有重复逻辑（超过5行相同代码应抽取）
- [ ] 共享工具函数放在正确的模块中
- [ ] 错误处理模式在整个模块中一致
- [ ] 日志记录模式一致
- [ ] API 响应格式一致

## 错误处理（权重15%）
- [ ] 所有异步操作有 try-catch 或 .catch()
- [ ] 错误没有被静默吞掉
- [ ] 错误信息描述清楚并包含上下文
- [ ] HTTP 错误使用正确的 tRPC 错误码
- [ ] 验证错误返回了具体的字段级信息

## 可维护性（权重15%）
- [ ] 复杂逻辑有注释说明为什么这么做
- [ ] 公开 API 有 JSDoc 注释
- [ ] 魔法数字和魔法字符串已抽取为常量
- [ ] 配置值已外部化（不硬编码在代码里）
- [ ] 没有无关联 Issue 的 TODO/FIXME/HACK

## 打分标准
- 10分：标杆级——可以作为团队参考代码
- 8-9分：优秀——有小改进空间
- 6-7分：合格——有些问题需要处理
- 4-5分：不达标——需要较大改进
- 1-3分：差——建议大幅重写
