# 安全审计检查清单（基于OWASP）

## A01：访问控制失效
- [ ] 所有接口都要求认证（除非明确标记为公开）
- [ ] 每个受保护资源都有权限检查
- [ ] 没有 IDOR 漏洞（用户只能访问自己的资源）
- [ ] CORS 配置正确（生产环境不使用通配符 *）
- [ ] tRPC 使用 protectedProcedure 而非 publicProcedure（除非确认公开）

## A02：加密失败
- [ ] 没有硬编码的密钥、密码或 token
- [ ] 敏感数据在存储时加密（PII、token）
- [ ] 所有外部通信使用 HTTPS
- [ ] 密码用 bcrypt 或 argon2 哈希（不是 MD5/SHA1）
- [ ] API 密钥从环境变量加载（通过 packages/env）
- [ ] 以 VITE_ 开头的环境变量没有包含敏感信息

## A03：注入
- [ ] Drizzle 查询使用参数化方式（内置的，检查有没有绕过）
- [ ] 用户输入在处理前已验证（Zod schema）
- [ ] 查询中没有字符串拼接
- [ ] 没有把用户输入传入 exec/spawn（命令注入）
- [ ] 没有把用户输入用于文件路径（路径遍历）

## A04：不安全设计
- [ ] 认证接口有频率限制
- [ ] 失败次数过多后有账户锁定或延迟机制
- [ ] 考虑了业务逻辑滥用场景
- [ ] 输入大小有限制

## A05：安全配置错误
- [ ] 生产配置中没有 debug 模式
- [ ] 错误信息不暴露堆栈跟踪给用户
- [ ] 不必要的功能和接口已禁用
- [ ] 安全头已设置（CSP、HSTS、X-Frame-Options）

## A07：认证失败
- [ ] 会话管理实现正确
- [ ] 有 token 刷新机制
- [ ] 登出时服务端会话失效
- [ ] 敏感操作要求二次验证（如果适用）

## A08：数据完整性
- [ ] 依赖没有已知的严重漏洞（pnpm audit）
- [ ] 没有使用已废弃的包
- [ ] 第三方包来源可信

## 打分标准
- 10分：铜墙铁壁——所有检查全部通过
- 7-9分：稳固——有小问题但没有严重漏洞
- 4-6分：令人担忧——有严重问题需要修复
- 1-3分：危险——需要立即补救
